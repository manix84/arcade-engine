# 💬 Support

Arcade Engine is a personal project, so support is best-effort.

## 🧭 Start Here

- Use [README.md](README.md) for package installation, imports, and the module
  map.
- Use [src/README.md](src/README.md) to understand the source modules exported
  by the package.
- Use [src/stories/README.md](src/stories/README.md) to understand the
  Storybook demos and what each section is proving.
- Use the nearest subsection README when working inside helpers, sound, ticker,
  graphics, stories, or tests.

## 🐛 Bugs

For bugs, open an issue with:

- What happened.
- What you expected.
- Steps to reproduce.
- Browser and operating system.
- Package version or commit SHA.
- Storybook story name, if the issue is visible in a demo.
- Test output or console errors.
- A minimal code example when possible.

## 💡 Feature Requests

Feature requests are easiest to evaluate when they describe:

- The game problem being solved.
- The API shape you expect.
- Why existing modules do not cover the use case.
- Whether the behavior belongs in the engine, a Storybook demo, or both.
- Any compatibility concerns with the package direction.

## 🛡️ Security Issues

Do not open public issues for security problems. Follow
[SECURITY.md](SECURITY.md).

## 📚 Documentation

Documentation fixes are welcome, especially when they clarify package status,
public APIs, Storybook demos, or test expectations.
